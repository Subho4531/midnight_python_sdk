# Changelog

All notable changes to the Midnight Python SDK will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.1] - 2024-04-11

### Added
- Comprehensive PyPI-ready documentation
- Installation verification command (`midnight verify`)
- Enhanced CLI with full command reference
- Type-safe Python bindings for smart contracts
- Privacy-preserving AI inference capabilities
- Complete transaction lifecycle management
- Real-time blockchain monitoring
- Auto-generated contract bindings from Compact files

### Changed
- Improved README with detailed usage examples
- Enhanced pyproject.toml with proper PyPI metadata
- Consolidated documentation into single comprehensive guide
- Updated CLI command structure for better usability

### Removed
- Unnecessary documentation files for cleaner PyPI package
- Development-specific files not needed for distribution
- Redundant markdown files and guides

### Fixed
- Package structure for proper PyPI distribution
- CLI command registration and imports
- Type hints and error handling

## [0.1.0] - 2024-04-10

### Added
- Initial release of Midnight Python SDK
- Core blockchain interaction capabilities
- Smart contract deployment and interaction
- ZK-SNARK proof generation
- Wallet integration with cryptographic signing
- GraphQL indexer integration
- Docker-based local development environment
- Comprehensive CLI tool
- AI inference with zero-knowledge proofs
- Real-time transaction monitoring
- Auto-generated Python bindings from Compact contracts

### Features
- **MidnightClient**: Main client for blockchain interaction
- **Wallet Operations**: Key management and transaction signing
- **Contract Management**: Deploy and interact with smart contracts
- **ZK Proofs**: Generate and verify zero-knowledge proofs
- **AI Integration**: Privacy-preserving machine learning
- **CLI Tool**: Complete command-line interface
- **Type Safety**: Full TypeScript-style type hints
- **Real-time Updates**: WebSocket-based event monitoring

### Supported Networks
- Local development (`undeployed`)
- Testnet integration
- Mainnet preparation (coming soon)

### Documentation
- Complete API reference
- Usage examples and tutorials
- CLI command reference
- Development setup guide
